WHERE条款支持

WHERE条款支持

该库支持创建非常灵活的WHERE子句。可以将WHERE子句添加到DELETE，SELECT和UPDATE语句中。WHERE子句也可以单独用于其他手工编码的SQL。

¶简单的WHERE子句

最简单的WHERE子句具有以下形式：

SelectStatementProvider selectStatement = select(count())
         .from(simpleTable)
         .where(id, isEqualTo(3))
         .build()
         .render(RenderingStrategies.MYBATIS3);

该库附带了多种条件，可在WHERE子句中使用，包括“ in”，“ like”，“ between”，“ isNull”，“ isNotNull”以及所有常规比较运算符。例如：

SelectStatementProvider selectStatement = select(count())
         .from(simpleTable)
         .where(id, isBetween(3).and(6))
         .build()
         .render(RenderingStrategies.MYBATIS3);
         
 SelectStatementProvider selectStatement = select(count())
         .from(simpleTable)
         .where(id, isIn(3,4,5))
         .build()
         .render(RenderingStrategies.MYBATIS3);
         
SelectStatementProvider selectStatement = select(count())
         .from(simpleTable)
         .where(id, isNotNull())
         .build()
         .render(RenderingStrategies.MYBATIS3);

¶复杂的WHERE子句

条件几乎可以任意组合进行“与”和“与”运算。例如：

SelectStatementProvider selectStatement = select(count())
           .from(simpleTable, "a")
           .where(id, isGreaterThan(2))
           .or(occupation, isNull(), and(id, isLessThan(6)))
           .build()
           .render(RenderingStrategies.MYBATIS3);

¶子查询

大多数条件还支持子查询。例如：

SelectStatementProvider selectStatement = select(column1.as("A_COLUMN1"), column2)
           .from(table, "a")
           .where(column2, isIn(select(column2).from(table).where(column2, isEqualTo(3))))
           .or(column1, isLessThan(d))
           .build()
           .render(RenderingStrategies.MYBATIS3);

¶独自站在条款

如果您希望为语句的其余部分编写自己的SQL，则可以单独使用where子句支持。这样做可能有多种原因-主要是在该库不支持您要使用的某些SQL或MyBatis功能的情况下。一个很好的例子是该库当前不支持的分页查询。如果要使用独立的where子句，可以编写如下所示的mapper方法：

@Select({
      "select id, animal_name, brain_weight, body_weight",
      "from AnimalData",
      "${whereClause}"
  })
  @ResultMap("AnimalDataResult")
  List<AnimalData> selectByExample(WhereClauseProvider whereClause);

您可以构建一个独立的where子句，然后像这样调用您的映射器：

WhereClauseProvider whereClause = where(id, isNotBetween(10).and(60))
        .build()
        .render(RenderingStrategies.MYBATIS3);

List<AnimalData> animals = mapper.selectByExample(whereClause);

当该语句不需要其他参数并且不涉及表别名时，此方法效果很好。如果您还有其他需求，请参阅以下内容。

¶表别名

如果需要在生成的where子句中使用表别名，则可以使用TableAliasCalculator该类在render方法上提供表别名。例如，如果您有一个这样的映射器：

@Select({
    "select a.id, a.animal_name, a.brain_weight, a.body_weight",
    "from AnimalData a",
    "${whereClause}"
})
@ResultMap("AnimalDataResult")
List<AnimalData> selectByExampleWithAlias(WhereClauseProvider whereClause);

然后可以在render方法上为生成的WHERE子句指定别名，如下所示：

WhereClauseProvider whereClause = where(id, isEqualTo(1), or(bodyWeight, isGreaterThan(1.0)))
         .build()
         .render(RenderingStrategies.MYBATIS3, TableAliasCalculator.of(animalData, "a"));

 List<AnimalData> animals = mapper.selectByExampleWithAlias(whereClause);

您很可能将表别名与带有手工编码的联接一起使用，而表别名的使用更多。在这种情况下，您Map<SqlTable, String>可以为TableAliasCalculator提供一个，其中包含WHERE子句中涉及的每个表的别名。

¶处理多个参数

默认情况下，WHERE子句渲染器假定渲染的WHERE子句将是映射器方法的唯一参数。这并非总是如此。例如，假设您有一个类似的分页查询（这是HSQLDB语法）：

@Select({
     "select id, animal_name, brain_weight, body_weight",
     "from AnimalData",
     "${whereClauseProvider.whereClause}",
     "order by id",
     "OFFSET #{offset,jdbcType=INTEGER} LIMIT #{limit,jdbcType=INTEGER}"
 })
 @ResultMap("AnimalDataResult")
 List<AnimalData> selectByExampleWithLimitAndOffset(@Param("whereClauseProvider") WhereClauseProvider whereClause,
         @Param("limit") int limit, @Param("offset") int offset);

在此映射器方法中，有三个参数。因此，在这种情况下，有必要告诉WHERE呈现的参数名称，以使用for呈现的where子句。该代码如下所示：

WhereClauseProvider whereClause = where(id, isLessThan(60))
        .build()
        .render(RenderingStrategies.MYBATIS3, "whereClauseProvider");
        
List<AnimalData> animals = mapper.selectByExampleWithLimitAndOffset(whereClause, 5, 15);

请注意，该字符串whereClauseProvider既用作映射器@Param注释中的参数名称，又用作render方法中的参数名称。

render方法也有一个重写，它接受TableAliasCalculator和参数名称。
