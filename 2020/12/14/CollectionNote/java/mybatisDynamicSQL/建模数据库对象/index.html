建模数据库对象

数据库对象表示

MyBatis动态SQL可与代表关系表或视图的Java对象一起使用。

¶表或视图表示

该类org.mybatis.dynamic.sql.SqlTable用于表示数据库中的表或视图。一个SqlTable包含一个名称，以及SqlColumn代表表或视图中列的对象的集合。

SQL中的表或视图名称包含三个部分：

  1. 目录-是可选的，在Microsoft SQL Server外部很少使用。如果未指定，将使用默认目录-许多数据库只有一个目录
  2. 模式-是可选的，但经常指定。如果未指定，将使用默认架构
  3. 表格名称-必填

名称的典型示例如下：

  • "dbo..bar"-具有目录（dbo）和表名（bar）的名称。这对于SQL Server是典型的
  • "foo.bar"-具有模式（foo）和表名（bar）的名称。当您要访问不在默认模式中的表时，这在许多数据库中都是典型的
  • "bar"-仅带有表名（栏）的名称。这将访问默认目录和架构中的表或视图以进行连接

在MyBatis Dynamic SQL中，可以通过不同方式指定表或视图名称：

  1. 名称可以是常量String
  2. 该名称可以在运行时根据目录和/或模式供应商功能以及常量表名称进行计算
  3. 可以使用名称提供者功能在运行时计算名称

¶常量名称

当将SqlTable构造函数与单个String参数一起使用时，将使用常量名称。例如：

public class MyTable extends SqlTable {
    public MyTable() {
        super("MyTable");
    }
}

要么

public class MyTable extends SqlTable {
    public MyTable() {
        super("MySchema.MyTable");
    }
}

¶动态目录和/或架构名称

MyBatis动态SQL允许您动态指定目录和/或架构。这对于架构可能针对不同的用户或环境而更改的应用程序，或者使用不同的架构分片数据库的应用程序，非常有用。当您使用SqlTable接受一个或多个java.util.function.Supplier参数的构造函数时，将使用动态名称。

例如，假设您想基于系统属性的值来更改架构。您可以编写这样的类：

public class SchemaSupplier {
    public static final String schema_property = "schemaToUse";

    public static Optional<String> schemaPropertyReader() {
        return Optional.ofNullable(System.getProperty(schema_property));
    }
}

此类具有静态方法schemaPropertyReader，该方法将返回，Optional<String>其中包含系统属性的值。然后，您可以SqlTable像下面这样在构造函数中引用此方法：

public static final class User extends SqlTable {
    public User() {
        super(SchemaSupplier::schemaPropertyReader, "User");
    }
}

只要引用该表以呈现SQL，就会根据系统属性的当前值来计算名称。

有两个可用于动态名称的构造函数：

  1. 接受Supplier<Optional<String>>模式和String名称的构造函数。此构造函数假定目录始终为空或未使用
  2. 接受Supplier<Optional<String>>目录，Supplier<Optional<String>>架构和String名称的构造函数

如果您使用的是Microsoft SQL Server，并且想使用动态目录名称而忽略架构，则应使用第二个构造函数，如下所示：

public static final class User extends SqlTable {
    public User() {
        super(CatalogSupplier::catalogPropertyReader, Optional::empty, "User");
    }
}

下表显示了如何在所有供应商组合中计算名称：

目录供应商价值 模式供应商价值 名称 全限定名
“我的目录” “ MySchema” “ MyTable” “ MyCatalog.MySchema.MyTable”
<空> “ MySchema” “ MyTable” “ MySchema.MyTable”
“我的目录” <空> “ MyTable” “ MyCatalog…MyTable”
<空> <空> “ MyTable” “ MyTable”

¶全动态名称

MyBatis动态SQL允许您动态指定完整的表名。这对于使用不同的表对数据库进行分片的应用程序很有用，这些表代表整体的不同分片。当您使用SqlTable接受单个java.util.function.Supplier参数的构造函数时，将使用动态名称。

请注意，此功能应仅用于名称不同但在其他方面相同的表。

例如，假设您想根据系统属性的值更改名称。您可以编写这样的类：

public class NameSupplier {
    public static final String name_property = "nameToUse";

    public static String namePropertyReader() {
        return System.getProperty(name_property);
    }
}

此类具有静态方法namePropertyReader，该方法将返回，String其中包含系统属性的值。然后，您可以SqlTable像下面这样在构造函数中引用此方法：

public static final class User extends SqlTable {
    public User() {
        super(NameSupplier::namePropertyReader);
    }
}

只要引用该表以呈现SQL，就会根据系统属性的当前值来计算名称。

¶列表示

该类org.mybatis.dynamic.sql.SqlColumn用于表示表或视图中的列。一个SqlColumn总是与相关SqlTable。在其最基本的形式中，SqlColumn该类包含一个名称和SqlTable对其关联的引用。需要使用表引用，以便可以在呈现阶段将表别名应用于列。

该SqlColumn会在SQL基础上，呈现RenderingStrategy适用于SQL语句。通常，渲染策略会生成一个字符串，该字符串表示您正在使用的任何SQL引擎中的参数标记。例如，MyBatis3参数标记的格式为“＃{some_attribute}”。默认情况下，所有列均以相同的策略呈现。该库提供适用于多个SQL执行引擎（包括MyBatis3和Spring JDBC模板）的呈现策略。

在某些情况下，有必要重写特定列的呈现策略-因此SqlColumn该类支持为列指定一个呈现策略，该列将覆盖应用于语句的呈现策略。PostgreSQL是这种用例的一个很好的例子。在该数据库中，插入或更新JSON字段时需要将字符串“ :: jsonb”添加到准备好的语句参数标记中，而其他字段则不需要。基于列的渲染策略可以实现这一点。

该SqlColumn班有是SQL渲染有用的附加可选属性-尤其是在MyBatis3。这些包括：

  • 该java.sql.JDBCType列。这将呈现到MyBatis3兼容的参数标记中-这有助于选择类型处理程序以及插入或更新具有null功能的字段
  • 包含类型处理程序的字符串-类型处理程序别名或类型处理程序的完全限定类型。这将呈现到MyBatis3兼容参数标记中

如果未使用MyBatis3，则无需指定JDBC Type或类型处理程序，因为其他呈现策略会忽略这些属性。

最后，SqlColumn该类具有指定用于不同SQL语句的列别名或排序顺序的方法。

我们建议使用一种使用方式来创建表和列对象，该方式为使用提供了很大的灵活性。有关完整示例，请参见“快速入门”页面。
